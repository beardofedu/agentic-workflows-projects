# agentic-workflows-projects

A demonstration repository showcasing how **agentic workflows** can be used with GitHub Projects (sprint boards) to identify work items that could be delayed and analyze downstream effects.

## 🛍️ TechMart Demo App

A fictional tech store built as a static web app, hosted on GitHub Pages. The app is intentionally incomplete — certain features are stubbed out to simulate a real sprint backlog.

**[View the live demo →](https://beardofedu.github.io/agentic-workflows-projects/)**

### Features Shipped
- Product catalog with category filtering
- Add-to-cart with count tracking
- Responsive layout (mobile-friendly)
- Hero banner, deals section, about page

### Stubbed Features (Backlog)
Each stub is tagged with a sprint and ticket number — perfect for sprint board demos:

| Feature | Ticket | Sprint |
|---|---|---|
| User Authentication | AUTH-001/002 | Sprint 2 |
| Shopping Cart Checkout | CART-003 | Sprint 1 (in-progress) |
| Promotions / Deals Engine | PROMO-007/008 | Sprint 3 |
| Order History | ACC-012 | Sprint 4 |
| Wishlist | ACC-005 | Sprint 2 |
| Help Center & Contact | SUP-001/002 | Sprint 4–5 |
| Returns Portal | OPS-009 | Sprint 4 |

## Running Locally

Just open `index.html` in a browser — no build step required.

## GitHub Pages Setup

Enable GitHub Pages in repository **Settings → Pages → Source: Deploy from branch → `main` / `root`**.
